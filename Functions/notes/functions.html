Functions

Defining & Calling Functions

Functions can be defined as a function declaration or function expression or an arrow function in ES6/ECMAScript 2015: 

      
        Example of a function declaration:
        function helloWorld() {
          console.log(`Hello There!`)
        }

        Example of a function expression:
        const helloWorld = function() {
          console.log(`Hello There!`)
        }

        Example of an arrow Function:
        const helloWorld = () => {
          console.log(`Hello There!`)
        }

After functions are defined they need to be called in order to run. The above functions can be called with helloWorld();

Parameters

Parameters are variables passed into the function as placeholders for when the function is called when they are then called arguments. Basically parameters are used during the defining of a function and arguments are used when the function is called 

      
        Example defining and calling a function with parameters:
        const nameOfFunction = function(parameter1, parameter2) {
          return parameter1 + parameter2;
        }

        nameOfFunction(2, 4);
        // Value would be 6

Return vs. Console.Log

Console.log prints or logs values to the console whereas Return is used to store a value to be reused.

Parameters & Arguments

Multiple Parameters

There can be an unlimited amount of parameters added to a function and the order at which they are added is important.

Protecting Against Missing Arguments

This can be accomplished with an if statement to check when a parameter is undefined however the newer way is with ES6 Default Parameters 

      
        Example of default parameters:
        const logPet = function(petName, petType = `dog`) {
          console.log(`${petName} is a ${petType}.`);
        }
        logPet(`Snickers`); // Snickers is a dog.
        logPet(`Coco`, `cat`); // Coco is a cat.

Scope

Variables declared outside of a function are global scoped, which means they can be accessed r over-written from anywhere within the code. In contrast, variables declared within the function have a local scope and are only available to that function.
